# Static Factory Method

  • Java constructors are the default mechanism for getting fully inilialized class instance
  • But in few specific use cases, it's preferable to use static factory methods for achieving the same result
  • When we have to create more than once constructor then we can go with Static Factory Method

Advantages of Static Factory Methods Over Constructors

  • Constructors don't have meaningful names, Static factory methods can have meaningful names
  • Static factory methods can return the same type that implements the method(s), a subtype, and also primitives, so they offer a more flexible range of returning types
  • Static factory methods can encapsulate all the logic required for pre-constructing fully initialized instances, so they can be used for moving this additional logic out of constructors. This prevents constructors from performing further tasks, others than just initializing fields
  • Static factory methods can be controlled-instanced methods, with the Singleton pattern being the most glaring example of this feature

Example

There are several examples of user of static factory methods. Here below is the example of string class. 

        
    
          
  • Creating object of String using constructor
    • 
          String value = new String("Baeldung");

          
  • Creating object of String using static factory methods
    • 
          String value1 = String.valueOf(1);
          String value2 = String.valueOf(1.0L);
          String value3 = String.valueOf(true);
          String value4 = String.valueOf('a');
        

      
        Custom Static Factory Method

        public final class Product { 
          private int price; 
          private String description; 
 
          private Product(int price,String description) { 
            this.price = price; 
            this.description = description; 
          }
          
          // Static factory method
          public static Product getNewProduct(int price,String description) { 
            return new Product(price,description); 
          }
        }
